# 產品需求文件 (PRD)：任務管理系統

## 1. 專案概述
- **背景與動機**：現代人生活與工作步調緊湊，常面臨多項任務交錯的情況，缺乏一個有效記錄並追蹤事項的工具。本專案旨在提供一個輕量、簡單且直覺的任務管理系統，協助學生與工作者有效管理日常待辦事項，提升個人生產力。
- **目標用戶**：學生及工作者。
- **核心價值主張**：提供一個無需複雜設定、隨開即用的個人任務管理系統，讓使用者能專注於任務本身的執行，而非花時間學習管理工具。

## 2. 功能需求
- **主要功能列表**：
   1. 新增任務
   2. 編輯任務內容
   3. 刪除任務
   4. 標記任務為完成狀態
   5. 將已完成的任務以刪除線劃掉（視覺回饋）
   6. 瀏覽與過濾任務（例如：查看所有、未完成、已完成）
   7. 設定任務截止日期

- **使用者故事 (User Stories)**：
   - 作為 **學生或工作者**，我希望 **能夠新增任務**，以便 **記錄我需要完成的工作項目**。
   - 作為 **學生或工作者**，我希望 **能夠編輯任務內容**，以便 **在任務細節或需求變更時保持資訊正確**。
   - 作為 **學生或工作者**，我希望 **能夠刪除任務**，以便 **移除錯誤輸入或已經不再需要執行的待辦事項**。
   - 作為 **學生或工作者**，我希望 **能夠將任務標記為完成**，以便 **清楚知道哪些事情已經做完並獲得成就感**。
   - 作為 **學生或工作者**，我希望 **已完成任務能自動被加上刪除線劃掉**，以便 **在視覺上快速區分任務狀態，帶來直觀的完成紀錄與成就感**。
   - 作為 **學生或工作者**，我希望 **能夠瀏覽並根據狀態過濾任務**，以便 **快速對焦目前需要處理的清單**。
   - 作為 **學生或工作者**，我希望 **能為任務設定截止日期**，以便 **依據輕重緩急合理安排我的時間**。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：Python Web 框架 Flask
  - 前端模板與渲染：Jinja2
  - 資料庫：SQLite (適合輕量級應用與開發階段快速迭代)
- **效能考量**：
  - 系統需具備快速的響應時間，確保頁面載入速度與各項操作反應在可接受的範圍內，提供流暢的使用體驗。
- **安全考量**：
  - 基本防護機制，如防止 SQL Injection 及 XSS 防護，確保系統與使用者輸入的安全。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須要有 - 核心 MVP)**：
  - 新增任務
  - 列表顯示所有任務
  - 編輯任務
  - 刪除任務
  - 標記任務完成狀態
  - 視覺回饋：將完成的任務以刪除線劃掉
- **Should Have (應該要有 - 強化可用性)**：
  - 依狀態過濾任務（全部 / 未完成 / 已完成）
  - 設定任務截止日期
- **Nice to Have (加分項目 - 未來優化)**：
  - 任務分類 / 標籤功能
  - 優先級設定（高/中/低）
  - 漂亮且現代化的 UI 設計（包含深色模式支援）
