# 個人記帳簿系統 PRD

## 1. 專案概述
- **背景與動機**：為了幫助學生與工作者有效管理個人財務，清楚掌握資金流向與餘額，避免不必要的浪費。
- **目標用戶**：學生、一般工作者。
- **核心價值主張**：提供簡單、直覺的記帳體驗，並結合視覺化的圖表分析，讓用戶能夠一目了然目前的收支與財務狀況。

## 2. 功能需求

以下列出系統的主要功能與對應的使用者故事：

1. **記錄收入**
   - *作為*一名學生或工作者，*我希望*能夠記錄我的各項收入（如零用錢、打工薪水、薪資等），*以便*清楚知道目前手上有多少進帳。
2. **記錄支出**
   - *作為*一名學生或工作者，*我希望*能夠分類記錄我的每筆支出，*以便*追蹤錢都花到哪裡去。
3. **顯示餘額統計**
   - *作為*一名使用者，*我希望*系統能自動計算並顯示目前的總餘額（總收入減去總支出），*以便*我能隨時評估是否需要節制開銷。
4. **顯示支出分類圓餅圖**
   - *作為*一名使用者，*我希望*能看到以圓餅圖呈現的支出分類比例（例如飲食、交通、娛樂等），*以便*我一眼看出主要的開銷來源，進而調整消費習慣。
5. **檢視收支歷史紀錄**
   - *作為*一名使用者，*我希望*能以列表的形式檢視過去所有的收支明細，*以便*在需要時回顧或核對特定的消費項目。

## 3. 非功能需求
- **技術限制**：預計採用 Flask + Jinja2 + SQLite 進行開發。
- **效能考量**：系統需保持輕量，頁面載入及資料儲存的回應時間應小於 1 秒。
- **安全考量**：需防範基本的 SQL Injection 攻擊，確保使用者資料儲存的安全性。

## 4. MVP 範圍

為了確保專案能如期上線並驗證核心價值，初期版本（MVP）的功能範圍劃分如下：

- **Must Have（必須有）**：
  - 記錄收入功能
  - 記錄支出功能
  - 顯示當前餘額統計
  - 檢視收支歷史明細列表
- **Should Have（應該有）**：
  - 支出分類方向的圓餅圖顯示
- **Nice to Have（可以有）**：
  - 自訂支出/收入分類類別
  - 匯出收支紀錄為 CSV 報表
